git reset --soft COMMITHASH

The --soft option is useful if you just want to go back to a previous commit, but keep all of your changes. Committed changes will be uncommitted and staged, 
while uncommitted changes will remain staged or unstaged as before.

In the last lesson, we undid a commit but kept the changes. We don't want to keep the changes to titles.md. Here's how to reset those changes:

git reset --hard COMMITHASH

The --hard flag makes your working directory and staging area match the specified commit exactly, discarding any local changes. 
This is useful if you just want to go back to a previous commit and discard all the changes

However, if you used git reset --hard to undo committing that file, it would be deleted for good.
